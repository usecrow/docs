## Server-Side Tools

These are tools that run on a server. The agent calls them, they execute, and results come back.

| Type | Best for | Setup |
|------|----------|-------|
| [OpenAPI](/openapi) | Your REST API with a spec file | Upload OpenAPI/Swagger spec |
| [Server-Side MCP](/server-side-mcp) | Your backend via MCP protocol | Run an MCP server, connect URL |

### When to Use OpenAPI

- You have a REST API with an OpenAPI/Swagger spec
- You want simple endpoint calls (check orders, create tickets)
- You need API Key, Bearer Token, or JWT Forward auth

### When to Use Server-Side MCP

- You want richer tool definitions with complex logic
- Your API doesn't have a clean OpenAPI spec
- You want live tool discovery (add tools without re-uploading a spec)
- You need to forward end-user identity to your server

<Note>
OpenAPI and Server-Side MCP are **mutually exclusive** per product. Pick one.
</Note>

## Client-Side Tools

Tools that run in the user's browser. The agent calls them, the widget executes them locally.

| Type | Best for |
|------|----------|
| [Client-Side Tools](/client-side-tools) | Browser actions (navigation, UI updates, local data) |
| [Tool Renderers](/tool-renderers) | Custom UI for tool results (charts, tables) |

## External MCP Servers

Third-party MCP servers that you connect to (not your own backend).

| Type | Best for |
|------|----------|
| [MCP Servers](/mcp-servers) | Third-party services (Notion, databases, etc.) |

---

## Example Actions

**Server-Side (OpenAPI or MCP):**
- Check order status
- Create support tickets
- Look up customer info
- Process refunds

**Client-Side:**
- Navigate to a page
- Open a modal
- Copy text to clipboard

**External MCP:**
- Search Notion pages
- Query external databases
